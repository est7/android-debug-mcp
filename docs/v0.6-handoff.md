- inside-default badge placement + outside-fallback,IHDR-pre-decode bomb guard
- soft-degrade only on `AnnotateError` + `CollectElementsError`,其余 IO / unknown
  propagate

### Tier 1 ⚡ 必做

#### #1 v2-G.1 Block B · `previewForAgent` 单条 record 截断

- **痛点**:Poppo lang.json 单条 ~155k tokens,Block A 治频率,Block B 治体积。**最大 token 暴击**。
- **lock 源**:[`v2/profile-and-evidence.md`](./v2/profile-and-evidence.md) § Known 技术债 已留款
- **方向**:`EvidenceSource.previewForAgent(record) → {preview, truncated, fullSizeBytes}`,read tools 默认 preview,`fullRecords:true` 显式付代价
- **同包**:Block A 软 bypass(`tsMsRange:{from:0}`)收紧(`from`/`to` 强制 + window-size cap),设计同时做
- **估时**:1.5-2 天 + codex 多轮 grill
- **影响面**:`server/src/profile/types.ts`(EvidenceSource interface 新方法)+ 所有 source 实现(`poppo_http/source.ts` 等)+ `server/src/mcp/tools/search_evidence.ts` 与 `extract_evidence_context.ts`(默认走 preview)+ schema 扩 `fullRecords?: boolean`
- **codex grill 预期**:contract 大动,**预 4-5 轮 STOP**(参考 v2-F.1 设计 5 轮 STOP 经验)
- **首步**:**先写 `docs/v2/preview-for-agent.md` 设计 lock**,grill 通过再动代码

#### #2 v2-F.3 · `list_elements` filter + pagination

- **痛点**:`list_elements` 一次返 60-80 元素 ~15-40 KB,v0.5.0 annotate 把它塞进 capture 后**单次 token cost 翻倍**
- **方向**:加 `{clickableOnly?, classContains?, inViewport?, limit?, cursor?}` input,**同 schema 同步加到 capture.annotateElements path**(避免两 tool drift)
- **估时**:0.8-1 天
- **影响面**:`server/src/mcp/tools/list_elements.ts` 扩 input schema + handler post-filter;`server/src/mcp/tools/capture.ts` annotate 分支同样过滤;新 fixture 测试
- **codex grill 预期**:1-2 轮(schema 改动不大,可能在 cursor 形态上 push)
- **未决问题**(grill 时一并定):
  - filter 字段命名:`classContains` vs `kindLike` vs ...
  - pagination cursor 形态(对齐 evidence cursor base64?或简单 offset?)
  - `inViewport` 语义(完全 ⊂ vs 相交)
  - 与 v2-F.2 (c) `annotateElementIds` 合一还是并存
- **首步**:看 v2-F.0 element-interaction.md 现有 design 段,确认 filter 形态不破 Q4 design lock
