## 当前 diff scope

- **v2-A.1** — `android_debug_map_ui_node_to_source`
  - 新增 `minConfidence?: "medium" | "high"` 与 `top?: number`。
  - `minConfidence` 按整体 tiered confidence verdict gate candidate list;不是
    0..1 numeric score,也不是 per-candidate confidence。
  - verdict 低于阈值时返回原 `confidence/reason/signals` +
    `candidates: []` + `warnings:["confidence_below_min"]`;不硬错、不 silent
    fallback 到 top-1。
  - 事件审计写入 `minConfidence` / `top` / `warnings`。

- **v2-G.2** — `android_debug_search_logs`
  - 原 tool 扩参,不新增 MCP tool。
  - 新增 aggregation mode:`count:true + groupBy:"tag"|"level"|"pid" + top?`。
  - count mode 返回 `entries: []`,并附
    `{groupBy, counts:[{group,count}], groupsTotal, otherCount}`。
  - 保留 no-fetch-all narrowing gate:`count:true` 不能替代 `query` / `level` /
    `sinceTs` / mark / `tags`。
  - count mode 不接受 `cursor`。
  - `errorType` 未做;等 stable parser/classifier,避免误导 taxonomy。

- **1.1-F** — lock test hermeticity
  - `getLocksRoot()` 支持 `ANDROID_DEBUG_MCP_LOCKS_ROOT`。
  - Vitest setup 注入 per-process tmp dir;production 无 env 时仍走
    `~/.android-debug-mcp/locks`。

- **Docs / hygiene**
  - `docs/backlog.md` 同步实际完成状态,删除重复 v2-G heading。
  - `docs/v2/source-mapping.md` 同步 v2-A.1 public contract。
